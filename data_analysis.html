Data Quality¶

For quick exploratory data analysis to spot check for data quality issues, I like to use the dtale package.¶

In [3]:
dtale.show(transactions)
Out[3]:




















Using the dtale package, we can quickly see some of the issues in the transaction dataset.¶
1) There are 13 rows in the transaction file that have been duplicated.¶


2) Looking at a QQ plot, we see three outlier amounts that may be data quality issues and warrant further investigating (also highlighted in red above). The highest value in particular is almost one order of magnitude larger thanthe rest of the amounts in the data set.¶


Based on these findings, a quick data quality recommendation for the client would be to add some basic data quality checks to the process that generates the files they send us. Duplicates rows and outlier values are attributes that are relatively easy to detect. Other suggestions for the client would be to standardize the naming of their filenames and column names to a uniform naming convention.¶
